## نظرة عامة على المسار الكامل

```
المستخدم يختار صورة
        ↓
[الواجهة] قراءة كـ base64 للمعاينة الفورية
        ↓
[الواجهة] رفع الملف الأصلي للخادم عبر FormData
        ↓
[الخادم — Multer] استقبال في الذاكرة (memoryStorage)
        ↓
[الخادم — processImage()]
   HEIC? → تحويل لـ JPEG أولاً
   sharp → تدوير تلقائي → تصغير → ضغط → حفظ
        ↓
[الخادم] إرجاع URL نسبي مثل: /api/uploads/cars/1234.jpg
        ↓
[الواجهة] تخزين URL في قائمة الصور للإعلان
        ↓
[عند النشر] إرسال مصفوفة URLs مع بيانات الإعلان لقاعدة البيانات
```

## 1. استقبال الصور على الخادم (Backend)

### المكتبات المستخدمة
| المكتبة | الدور |
|---|---|
| `multer` | استقبال ملفات multipart/form-data في الذاكرة |
| `sharp` | تصغير + تدوير + ضغط الصور |
| `heic-convert` | تحويل صور iPhone (HEIC) لـ JPEG على الخادم |

### إعدادات الاستقبال
```
الحجم الأقصى : 20 ميجابايت (لأن كاميرات Android/iOS تنتج ملفات كبيرة)
الصيغ المقبولة: JPEG · PNG · GIF · WebP · HEIC · HEIF
التخزين      : في الذاكرة (memoryStorage) ثم المعالجة وحفظ على القرص
```

### خطوات معالجة الصورة (`processImage`)

```
1. HEIC/HEIF؟ → heic-convert → JPEG buffer (جودة 82%)
           ↓
2. sharp.rotate()
   → تصحيح الاتجاه تلقائياً من بيانات EXIF
     (يمنع ظهور الصور بشكل معكوس من موبايل)
           ↓
3. sharp.resize({ width: 1600, withoutEnlargement: true })
   → تصغير العرض لـ 1600px كحد أقصى
   → إذا كانت الصورة أصغر من 1600px تُترك كما هي
   → الارتفاع يُحسب تلقائياً بنسبة الأصلية
           ↓
4. sharp.jpeg({ quality: 82, progressive: true })
   → تحويل لـ JPEG بجودة 82%
   → progressive: تُحمّل الصورة تدريجياً (أسرع في التصفح)
           ↓
5. حفظ بصيغة: uploads/{folder}/{timestamp}.jpg
   مثال: uploads/cars/1774359499404.jpg
           ↓
6. إرجاع URL: /api/uploads/{folder}/{timestamp}.jpg
```

**Fallback:** إذا فشل sharp وكان الملف JPEG أصلياً → يُحفظ مباشرة بدون معالجة.

## 3. رفع الصور في الواجهة (Frontend)

### الوضع العام (add-listing، rental-cars)

```javascript
// 1. المستخدم يختار الصورة من input[type=file]
// 2. قراءة كـ base64 للمعاينة الفورية
const reader = new FileReader();
reader.onload = (e) => setPreview(e.target.result); // base64

// 3. رفع الملف الأصلي للخادم
const formData = new FormData();
formData.append("image", file);
const res = await fetch("/api/upload-image?folder=...", { method: "POST", body: formData });
const { url } = await res.json();
// url = "/api/uploads/listings/1234.jpg"
setImages(prev => [...prev, url]);
```

### وضع كل شيء / Marketplace (يدعم Capacitor Android)

```javascript
// قراءة كـ base64 بدلاً من blob URL
// (blob URLs لا تعمل في WebView الأندرويد)
const base64 = await readFileAsBase64(file);
setPreviewImages(prev => [...prev, base64]); // عرض فوري

// HEIC → JPEG على الجهاز قبل الرفع (مكتبة heic2any)
if (file.type.includes("heic")) {
  file = await heic2any({ blob: file, toType: "image/jpeg", quality: 0.85 });
}

// رفع للخادم والحصول على URL نهائي
const url = await uploadImage(file, "marketplace");
```

## 4. كيف تُعرض الصور في كل مرحلة

### أ) معاينة قبل النشر (داخل النموذج)

| القسم | الحاوية | طريقة العرض |
|---|---|---|
| add-listing (سيارات) | مربع `80×80 px` أو `96×96 px` | `object-cover` + قابلة للحذف |
| marketplace (كل شيء) | شبكة صور صغيرة | `object-cover` + زر × للحذف |
| rental-cars | شريط صور أفقي | `object-cover` + زر × |
| car-parts / junk-cars | مصغرات صغيرة | `object-cover` + زر × |

**ملاحظة:** المعاينة تعمل بصور base64 (قبل الرفع) أو URL من الخادم (بعد الرفع).

---

### ب) الكارت المصغر في القائمة (بطاقة الإعلان)

**CarCard** (السيارات في الشاشة الرئيسية والقسم):
```
نسبة العرض   : 4:3  (aspect-ratio: 4/3)
عرض الكارت  : ~300–400px → الصورة ~300×225px
طريقة التناسب: object-cover (تملأ الحاوية وتقطع الزوائد)
الكاروسيل   : أسهم يمين/يسار عند وجود أكثر من صورة
URL          : imgUrl(images[index]) — يدعم URL نسبي وAbsolute وNative
```

**ListingCard** (قطع غيار، خردة، إيجار، وظائف، عقارات، كل شيء):
```
نسبة العرض   : 4:3
طريقة التناسب: object-cover
الكاروسيل   : أسهم + نقاط مؤشر
بدون صورة   : أيقونة placeholder حسب نوع الإعلان
```

---

### ج) الصورة في الشاشة الرئيسية

```
السيارات المميزة : CarCard — aspect-[4/3] — object-cover
العقارات الجديدة : ListingCard — aspect-[4/3] — object-cover
الوظائف         : لا صورة (بطاقة نصية فقط)
طلبات الشراء    : لا صورة (نصي فقط)
```

---

### د) داخل صفحة القسم (قبل فتح الإعلان)

نفس `CarCard` أو `ListingCard` بالإعدادات نفسها — aspect-[4/3] — object-cover.

---

### هـ) صفحة الإعلان المفتوحة (التفاصيل)

| الصفحة | نسبة الصورة الرئيسية | شبكة المصغرات |
|---|---|---|
| `car-detail.tsx` | `16:9` أفقي | شريط أفقي أسفل الصورة — `object-cover` |


## 5. تحويل URL الصورة (`imgUrl`)

```typescript
// artifacts/syrian-car-market/src/lib/runtimeConfig.ts

export function imgUrl(url: string | null | undefined): string | undefined {
  if (!url) return undefined;

  // 1. URL كامل (http/https) أو base64 أو blob → يُستخدم مباشرة
  if (url.startsWith("http") || url.startsWith("data:") || url.startsWith("blob:"))
    return url;

  // 2. URL نسبي مثل "/api/uploads/cars/123.jpg"
  const p = url.startsWith("/") ? url : `/${url}`;

  // 3. في APK Android (Capacitor) → يُضاف "https://marklet.net" أمامه
  //    في الويب → يُستخدم كما هو (Vite يُوكّل /api → localhost:8080)
  return IS_NATIVE ? `https://marklet.net${p}` : p;
}
```

## 7. المشاكل المعروفة والقيود الحالية

| المشكلة | الوضع الحالي | الإصلاح المقترح |
|---|---|---|
| لا ضغط على الجهاز قبل الرفع | الصورة ترفع بحجمها الأصلي (حتى 20MB) | ضغط Canvas قبل الرفع → هدف ≤500KB |
| حد العرض 1600px فقط | مناسب للويب لكن كبير للموبايل | يمكن خفضه لـ 1200px |
| لا تحويل WebP | الخادم يحفظ JPEG فقط | إضافة `.webp({ quality: 80 })` بديلاً |
| Reels لا تستخدم `imgUrl()` | `poster` و`src` تستخدم URL نسبي مباشرة | ربما لا يعمل في Native APK |
| لا Zoom/Pinch في التفاصيل | المستخدم لا يستطيع تكبير الصور | مكتبة react-zoom-pan-pinch |
| لا ترتيب السحب والإفلات | ترتيب الصور ثابت كما رُفعت | Drag-and-drop reorder |
