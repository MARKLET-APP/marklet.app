# نظام النشر الموحد — LAZEMNI
## المرجع الرسمي للمطورين — v1.0 (2026-04-05)

---

## المبدأ الذهبي

> **كل إعلان يمر بنفس المسار دون استثناء:**
> نشر → إشعار "قيد المراجعة" → مراجعة الأدمن → إشعار القبول/الرفض → فتح الإعلان بضغطة

---

## دورة حياة أي إعلان

```
1. المستخدم يرسل الإعلان
        ↓
2. Toast فوري في الواجهة
   "✅ تم إرسال طلبك للمراجعة
    سيظهر في القائمة بعد موافقة الإدارة"
        ↓
3. إشعار في قاعدة البيانات (يظهر في قسم الإشعارات)
   "إعلانك قيد المراجعة ⏳"
        ↓
4. الأدمن يراجع من لوحة التحكم
        ↓
    ┌──────────────┬──────────────┐
  قبول              رفض
    ↓                ↓
5a. إشعار DB:      5b. إشعار DB:
    "✅ تمت          "❌ تم رفض
    الموافقة..."     إعلانك..."
    ↓                ↓
6a. Push Notif    6b. Push Notif
    للجهاز           للجهاز
    ↓
7. المستخدم يضغط الإشعار
        ↓
8. ينتقل مباشرة لصفحة الإعلان
```

## قالب الإشعار الموحد

### عند إرسال الإعلان (Frontend Toast)
```
العنوان: ✅ تم إرسال طلبك للمراجعة
الوصف:  سيظهر في القائمة بعد موافقة الإدارة
```

### عند إرسال الإعلان (DB Notification)
```
type: "marketplace_pending"
message: 'إعلانك "{اسم الإعلان}" قيد المراجعة ⏳ — سيتم مراجعته ونشره قريباً'
link: null
```

### عند القبول (DB + Push)
```
type: "approval"
message: 'تمت الموافقة على إعلانك "{اسم الإعلان}" ونشره على LAZEMNI'
link: "/{قسم}/{معرّف}" — يُفتح عند الضغط على الإشعار
Push title: "✅ تمت الموافقة على إعلانك"
Push tag: "{قسم}-approved-{id}"
```

### عند الرفض (DB + Push)
```
type: "rejection"
message: 'تم رفض إعلانك "{اسم الإعلان}". يمكنك تعديله وإعادة إرساله'
link: null
Push title: "❌ تم رفض إعلانك"
Push tag: "{قسم}-rejected-{id}"
```

## جدول قاعدة البيانات: notifications

```sql
id          serial PRIMARY KEY
user_id     integer NOT NULL         -- المستخدم المستهدف
type        text    NOT NULL         -- "approval" | "rejection" | "marketplace_pending" | "marketplace_order" | ...
message     text    NOT NULL         -- نص الإشعار بالعربية
link        text                     -- الرابط الداخلي للتطبيق (مثال: /marketplace/42)
is_read     boolean DEFAULT false    -- هل قرأه المستخدم؟
created_at  timestamp with timezone
```

---

## قواعد لا يجوز كسرها

1. **كل إعلان جديد** → يجب أن يُنشئ سجل في `notificationsTable` بـ `type: "marketplace_pending"`
2. **كل موافقة/رفض** → يجب أن يُنشئ سجل بـ `type: "approval"` أو `"rejection"` + يُرسل Push
3. **الموافقة فقط** → تحتوي على `link` يفتح الإعلان
4. **الرفض** → `link: null` (لأن الإعلان قد يُحذف أو يكون غير منشور)
5. **صفحة الإشعارات** → الضغط على إشعار يعلّمه مقروءاً ويفتح الـ `link` إن وجد
6. **نص Toast** دائماً → `"✅ تم إرسال طلبك للمراجعة"` وليس `"تم النشر بنجاح"`
